# NBC_Work9 – 멀티플레이 숫자야구 게임

## 프로젝트 개요
이 프로젝트는 Unreal Engine 5와 C++를 활용하여 구현한 멀티플레이 숫자야구(숫자 맞추기) 게임입니다.

서버가 비밀 숫자를 생성하고, 각 클라이언트는 채팅으로 숫자를 입력하여 Strike / Ball 판정을 받으며, 턴 기반으로 제한 시간 내 숫자를 맞히는 것을 목표로 합니다.
<br></br>

## 프로젝트 목표
- 언리얼 멀티플레이 기본 구조 이해
  - GameMode / GameState / PlayerController / PlayerState의 역할 분리 및 상호작용 구현
- RPC와 Replication 활용
  - Server/Client RPC, NetMulticast, Replicated 변수 등을 활용한 상태 동기화 연습
- 턴제 게임 로직 설계
  - 턴 전환, 제한 시간, 최대 추측 횟수 등 게임 규칙을 시스템화하는 경험 축적
- UI/UX 개선
  - 채팅 기반 인터랙션 및 HUD/위젯을 통해 게임 정보를 직관적으로 제공
<br></br>

## 게임 규칙 & 진행 방식
### 1. 숫자야구 규칙
- 서버가 1~9까지의 서로 다른 숫자 3개로 이루어진 비밀 숫자를 생성합니다.
- 플레이어는 채팅창에 3자리 숫자를 입력하여 추측합니다.
- 입력된 숫자는 서버에서 다음 조건을 만족할 때만 유효한 추측으로 처리됩니다:
  - 길이 3자리
  - 각 자리가 1~9 사이의 숫자
  - 중복 숫자 없음
- 서버는 비밀 숫자와 비교하여 Strike / Ball을 계산합니다.
  - 자리와 숫자가 모두 같으면 Strike
  - 숫자만 같고 위치가 다르면 Ball
  - 하나도 맞지 않으면 OUT

### 2. 승리 및 무승부 조건
- 한 플레이어가 3 Strike(3S0B)를 달성하면 해당 플레이어가 승리로 처리되며, 모든 플레이어에게 승리 메시지가 Notification UI를 통해 브로드캐스트됩니다.
- 모든 플레이어가 각자 최대 추측 횟수를 소진하면 게임은 무승부(Draw)로 종료됩니다.
<br></br>

## 핵심 기능 정리
### 1. 실시간 채팅 및 숫자 판정
- 채팅창에 입력한 메시지 중 3자리 숫자는 자동으로 추측으로 인식되고, 서버에서 Strike/Ball/OUT을 판정한 뒤 모든 클라이언트에게 결과를 브로드캐스트합니다.
  
### 2. 턴 기반 멀티플레이 진행
- GameMode가 플레이어 배열을 순회하면서 현재 턴을 관리하고, GameState를 통해 현재 턴 플레이어 정보를 Replicate하여 UI에 표시합니다.

### 3. 턴 제한 시간 시스템
- 각 턴마다 TurnDuration만큼 시간이 주어지며, 1초마다 감소합니다.
- 시간이 0이 되면 자동으로 턴이 넘어가며, 현재 플레이어의 시도 횟수가 증가합니다.

### 4. 플레이어 별 시도 횟수 관리
- PlayerState에 현재 시도 횟수/최대 시도 횟수를 저장하고, 채팅 메시지에 "Player1(1/3)" 형태로 보여줍니다.

### 5. 승리 / 무승부 판정
- 어떤 플레이어가 3 Strike를 달성하면 승리 메시지를 전체에게 노출
- 모든 플레이어의 시도 횟수가 최대치에 도달하면 무승부로 처리 후 알림
